GOAL
Build a polished, shareable browser 3D game: a 1/10-scale DIRT-OVAL RC racing game
modeled on the real Team Losi 22S Sprint (TLR 22 platform). It must be a real
deliverable — `npm run build` produces a static `dist/` folder that runs from any
static host with no dev tooling and no server-side dependencies. Work in a loop: each
milestone ends in a runnable build that you VERIFY on screen (screenshot or read sim
state) before moving on. TWO player car classes (winged sprint + dirt late model),
each with its own body, physics baseline, and independent career. TWO game modes
(Career/Sim championship + Arcade RC Pro-Am style). A single UNIFIED SETUP SCREEN sets
driver name + car class + game mode + sound on/off + AUTO-THROTTLE, then START (all
persisted to localStorage["rcdirtoval.*"]); both modes launch off a drag-strip light
tree (see GAME MODES).

TECH STACK (no game engine install)
- Babylon.js 7 (@babylonjs/core, @babylonjs/materials) imported À LA CARTE
  (e.g. "@babylonjs/core/Meshes/mesh"), not from the barrel, to keep the bundle small.
  Remember the SIDE-EFFECT imports: standard/PBR materials, shadow + physics scene
  components, and prepass + geometryBuffer for SSAO2.
- Havok physics via @babylonjs/havok (WASM), async-initialized before the first step.
- Vite 5 + TypeScript 5, strict (noUnusedLocals, noUnusedParameters, noImplicitReturns).
- No test runner, no linter. `npm run build` (tsc --noEmit then vite build) is the gate.
- Persistence in localStorage. Audio is procedural Web Audio (see AUDIO) — no files ship.

CORE DESIGN DECISION — THE VEHICLE IS CUSTOM AND KINEMATIC (NOT a Havok body)
Implement a custom RaycastVehicle that integrates its OWN planar velocity (slip-based
tire model with a friction circle), its OWN yaw (bicycle model + slip oversteer +
throttle-steer), and RAYCASTS the ground each step for ride height and banking alignment.
Use Havok ONLY for static track collision and the wheel/ground raycasts — never to move a
car. (Reason: Havok v2 applyForce on a dynamic body desyncs velocity from the mesh.) Tune
handling through a VehicleConfig/DEFAULT_CONFIG (one baseline per class) and the slip/grip
math. Car-to-car contact and wall limits are resolved POSITIONALLY (not physics) in a Field
class that also owns surface grip, tire wear, dust, and ROLLOVERS for the whole field.
A genuinely hard T-bone or wall slam (closing speed over a threshold) triggers a
realistic MULTI-AXIS ROLLOVER that ends with the car STUCK (immobile, no control)
until a marshal rights it; the player can also tap R to bail out.

CAR CLASSES (CarClass.ts)
Two classes the player picks on the unified setup screen; CAR_CLASSES/CAR_CLASS_LIST map each id
to its body builder + PRISTINE physics baseline (cloned per car). loadCarClass/saveCarClass
persist the pick (localStorage["rcdirtoval.class"], ?class=sprint|latemodel override). Field
takes a CarClassDef and builds every car via classDef.build. Careers are CLASS-KEYED
(loadCareer(cls)/saveCareer(c,cls) under rcdirtoval.career.<cls>, with a one-time migration of
an old single-class save into sprint). SetupPanel shows the class label.
  1. WINGED SPRINT CAR (Car.ts / DEFAULT_CONFIG) — light, twitchy, wing downforce, power-oversteer.
  2. DIRT LATE MODEL (LateModel.ts / LATE_MODEL_CONFIG) — heavy, planted, mechanical grip, no wing;
     a real 1:10 RC dirt late model: a LOW, WIDE, FULL-BODIED WEDGE with big fenders covering the
     wheels, a small narrow cab set back, tall SAIL PANELS, a wide flat RAKED SPOILER, and a low
     full-width nose + splitter; numbered-roundel door livery + roof number, mild stagger.
HANDLING KNOBS (VehicleConfig): besides grip/steer basics, two yaw "looseness" gains separate the
classes — slipSteer (how much a slide rotates the car / oversteer) and throttleSteer (how much
throttle rotates it through a corner / dirt power-steer). Sprint = loose (0.6/0.015); late model =
planted (0.42/0.009) + more tireGrip + no downforce. (mass + suspension are currently cosmetic in
the kinematic model — a real load-transfer pass is a known future improvement.)

GAME MODES (game/Mode.ts, game/Arcade.ts)
Two modes the player picks on the UNIFIED SETUP SCREEN (along with name/class/sound/auto-throttle).
loadMode/saveMode persist the pick (localStorage["rcdirtoval.mode"], ?mode=career|arcade override);
arcade run-state {round, continues, score} persists in localStorage["rcdirtoval.arcade"]. Changing
class or mode persists + reloads + auto-starts. finalize() in main.ts branches results by mode.
LIGHT-TREE START (BOTH MODES): replace the plain 3-2-1 text countdown with a drag-strip LIGHT TREE
(Screens.arcadeLightTree: staging dots -> three ambers -> GREEN, "GET READY"/"GO!"), firing the green
flag at the same ~2.4s moment. PERFECT-LAUNCH BOOST: hitting gas within ~350ms of green grants a brief
acceleration buff (main.ts).
  1. CAREER/SIM — the championship season (always-advance, points keyed by name). UNCHANGED by arcade.
  2. ARCADE (RC Pro-Am style, ArcadeManager) — defaults to the RC PRO-AM OVERHEAD CAMERA (the car
     stays centered while the track scrolls under it) and scatters oil/wet SLICK patches (any car:
     transient grip loss -> slides) on the oval. (The old box PICKUPS, collectible LETTERS, and BOOST
     STRIPS were REMOVED — only slicks remain.) A score accrues. ADVANCEMENT: must finish TOP-3 to
     advance; else burn one of ~3 CONTINUES (out of continues -> run resets). Arcade HUD (#arcadeHud:
     Score/Continues) shows only in arcade mode; expose window.__arcade for tests.
TEMP BUFF LAYER ON THE VEHICLE (RaycastVehicle.ts): applyBuff(kind:"grip"|"accel"|"top", mult, seconds),
grantImmunity(seconds), buffState() — ticked/decayed in update(), woven into the grip/accel/top-speed
math; immunity short-circuits triggerRollover. Behavior IDENTICAL when no buff is active (Career/Sim
physics untouched).

MARSHALS (Marshals.ts) — RIGGED + ANIMATED
6 marshals — 2 SIT in camp chairs at the two infield ENDS (one chair per end) and 4 STAND OUTSIDE
the track at the corners (evenly spread, turns 1-4), like a real RC dirt oval. When a car is
WRECKED (stuck upside down) OR STALLED (stopped / pointing the wrong way at near-zero speed while
green), the NEAREST AVAILABLE marshal (seated or standing) gets up/jogs across traffic to it and
PLACES IT BACK ON THE RACING LINE — upright, facing race direction, ready to continue
(vehicle.resetTo(pos, yaw) from track.project() nearest centerline + tangent), then returns to its
post and re-seats/stands. Procedural low-poly figures, each with a `seated` flag for the return
posture, VARIED looks (shirt/hair/skin/headwear/cap) so no two read identically. Figures are RIGGED:
buildPerson builds legs on hip+knee pivots and arms on shoulder pivots (rest pose reads like a plain
standing figure), exposed via a personRigs WeakMap; a marshal ANIMATES A JOG CYCLE (legs swing from
the hips, knees bend, arms counter-swing) while moving to/from an incident, returning to rest when
standing/seated/working. Static figures (spectators) leave the rig at rest. EXTREMITIES: every person
built by buildPerson (marshals + spectators), plus the FlagGirl and the LawnMower rider, has visible
SHOES, HANDS, and KNEE joints (animated via the rig pivots); world scale unchanged (~5.7u, feet y≈0).

FIELD SIZE: each race runs a RANDOM 8-12 cars (def.fieldSize = 8 + rand(0..4) per load); keep the
colour PALETTE (Field.ts) and DRIVER_NUMBERS (Career.ts) at least 12 entries long.

AUDIO (MotorSound.ts): pure procedural Web Audio (no file). PLAYER car: emulate a HIGH-REVVING
COMBUSTION SPRINT-CAR ENGINE — a rich, harmonically dense voice whose pitch/rev tracks speed+throttle,
plus a faint speed-scaled noise tire hiss — through a low-pass that OPENS with throttle (keep cutoff
audible). AI FIELD: a cheaper tier — setVoiceCount(n) makes one lightweight osc -> gain -> STEREO
PANNER voice per other car, and updateVoices(...) each frame pitches/pans/distance-fades them to the
active camera so the pack is a background, not a swarm. AudioContext starts suspended; resume() on the
first user gesture (autoplay policy), then update(throttle, speed) + updateVoices each physics step
only while racing; smooth with setTargetAtTime. PAUSING silences the engine. Mute with M / a HUD
button -> master gain 0 (mutes everything), persisted to localStorage["rcdirtoval.muted"]. Degrade to a
silent no-op if no AudioContext.

GAME SHAPE
- FOUR camera views (view enum incl. normal|incar|aerial|proam; ALWAYS starts at Track each race; ?view= override):
  - Driver-stand ("Track"): an elevated trackside RC vantage that smoothly follows the player's
    car all the way around — aims straight at it (panning into the corners), slides along the
    straight to keep it close, and telephoto-zooms as it runs to the far side.
  - In-car/cockpit (CockpitCamera.ts): a first-person view parented to the player car root, so it
    inherits heading/pitch/roll/bank; adds a subtle lean into corners + a faint speed shake/FOV;
    falls back to the external cam during a flip.
  - Aerial: a high overview of the whole oval.
  - RC PRO-AM OVERHEAD: a top-down chase where the PLAYER CAR STAYS CENTERED and the track scrolls
    under it (the Arcade default view).
  New cameras must join the default render pipeline AND the ssao pipeline for full post-FX parity.
  An upper-left button + the V key cycle the views; C still quick-toggles aerial. MANUAL ZOOM works
  in every view (mouse wheel / +/- keys / on-screen touch +/- buttons). A hidden ?photo mode locks
  a close rear-3/4 camera onto the player car (shares post-FX) to show off the bodywork.
- Game-flow state machine in a single entry point (main.ts): attract -> prerace -> racing -> finished.
- PAUSE MENU (P / a HUD ⏸ button): Resume / Restart / Main Menu — freezes the fixed-timestep
  accumulator, the race clock, and the engine sound, then resumes cleanly.
- BOOT/LOADING PROGRESS BAR + OPENING LOGO: the loading splash shows the SUPER JAY #32 racing badge
  (public/superjay-32.png) above a real app-build progress bar (fill + label) advanced in stages as
  the engine, Havok physics, and track/field build, then fades out; it CREDITS "Created by Aaron Blair".
- UNIFIED SETUP SCREEN (Screens.setup): a SINGLE start screen replaces the old class -> mode ->
  pre-race -> name chain. It sets DRIVER NAME (pre-filled with the saved name, default "Super Jay";
  title-cased, persisted, shown on the leaderboard), CAR CLASS (Winged Sprint / Dirt Late Model),
  GAME MODE (Career/Sim vs Arcade — see GAME MODES), SOUND on/off, and AUTO-THROTTLE on/off, then
  START. Every setting is PERSISTED + remembered for next time (localStorage["rcdirtoval.*"]);
  changing class or mode persists + reloads + auto-starts so the field/career rebuild. AI cars get
  STABLE random full names (one per grid slot) so the championship — points keyed by driver name —
  accumulates coherently across the season.
- AUTO-THROTTLE (persisted): when ON, the car runs FULL THROTTLE always and the ONLY input is
  steering (the touch GAS/BRAKE pedals are hidden); works on desktop + mobile.
- A flag girl at the start/finish line waves the green flag on the GO of the LIGHT-TREE start (see GAME
  MODES); a polished in-game driver's manual overlay opens from the title and pre-race screens.
- ATTRACT INTRO: on first open per tab session, play a cinematic "broadcast" reel — the whole field
  is AI-driven and a single CinematicCamera cuts between a slow crane orbit, a low trackside rise, a
  chase behind the hero car, and a flyby dolly. Overlay a title card; any click/keypress flags
  sessionStorage and reloads into the menu with a fresh grid. It must read like a video (HUD hidden).
- A `?demo` URL param skips attract+menu straight into a live race; `?round=N` (1-based) forces a
  career round; `?class=sprint|latemodel` forces a class; `?mode=career|arcade` forces a game mode;
  `?day`/`?night` force lighting. Combine.
- Fixed-timestep accumulator: FIXED = 1/60, up to 6 catch-up steps per frame. Each step:
  Field.update(dt, input, raceFraction).
- HUD: lap, position, interval gaps, lap time, best, speed (mph-scaled), tire %, track state,
  minimap. On-screen touch controls on phones (steer pad + gas/brake/reset; touch-action:none on
  each control + cancel multi-touch touchmove so iOS pinch-zoom can't scroll them off).

TRACKS (data-driven)
- A TrackDef holds: cornerRadius, straightLength, width, banking (radians), baseGrip, gripFalloff
  (per lap), rutIntensity, aiSkill, fieldSize, laps, dirtColor, difficulty, night (bool), and
  backdrop (one of mesas/forest/plains/city/dunes/badlands).
- An OvalTrack procedurally builds a COUNTER-CLOCKWISE banked stadium oval (2 straights + 2
  180-degree turns), plus walls, infield/outfield ground, banners, a GRASSED INFIELD filling the
  oval with a speedway logo laid flat and faded onto it (a bundled image asset, alpha-blended/matte),
  a UNIFORM BROWN PACKED-DIRT racing surface (contiguous full-width ribbons all one earthy brown —
  NO painted multi-shade groove bands; grip evolves per line in SurfaceModel, and the DRIVEN RACING
  GROOVE visibly DARKENS over a run as it gets rubbered/packed in),
  start/finish, and centerline helpers project()/sampleAt()/gridPose(). Put the START/FINISH LINE
  about THREE-QUARTERS of the way down the front stretch. Build trackside scenery (Scenery.ts): a
  drivers' stand DOUBLED IN LENGTH (with ~8 VARIED full-size spectators on the deck — built via the
  marshals' buildPerson/spectatorLooks — YAWED TO FACE THE TRACK (-x) with arms posed forward, each
  holding a procedural RC TRANSMITTER prop), a BANNER and a LIT BOOTH WINDOW on it, tailgate-party
  PICKUP TRUCKS (full real-world scale) parked behind the grandstand + along the east straight, 6
  LIGHT TOWERS (four corners + two mid-straight), a start/finish gantry, and a PER-ROUND THEMED
  BACKDROP (mesas/forest/plains/city/dunes/badlands — distinct silhouette + color, darker at night)
  with themed near vegetation. The 400m infield only covers +/-200m, so also lay a large WORLD FLOOR
  (~1700m, tinted per theme) under everything or half-buried sphere hills/dunes float over void.
- generateCareer() returns 15 progressively harder ovals: corner radius shrinks, straights lengthen,
  width narrows, grip drops + falls off faster, ruts and AI skill rise, laps and field grow. Each
  round gets a DISTINCT dirt color AND backdrop theme. Banking ramps from flat early ovals to
  high-banks (~0.20-0.24 rad) late. THE GAME RUNS AT NIGHT, GAME-WIDE (def.night = true forced in
  main.ts) — dark sky, crescent moon + starfield, lit lamp towers. The night look is the game's
  identity; DO NOT ship daytime racing. ?day is a dev-only preview override. Keep night bright enough
  that the cars read (don't crush them into shadow).

CAREER / CHAMPIONSHIP (Career/Sim mode — Arcade advancement differs, see GAME MODES)
- Points by finish (25,20,16,13,11,9,7,5,4,3,2,1). Standings, save/load in localStorage, PER CAR CLASS.
- RACE END: the race ENDS ONE LAP AFTER THE WINNER crosses the start/finish line (the field gets a
  clean final lap), not the instant the leader finishes. Rank finished cars by FINISH TIME.
- PROGRESSION: in Career/Sim the season ALWAYS advances to the next (harder) round from the results
  screen (no podium gate) — championship points still accrue by finish. The finale shows the champion.
  (Arcade instead gates advancement on a TOP-3 finish, spending a continue otherwise — see GAME MODES.)
- EARLY-CAREER SPEED BOOST (main.ts): on career levels 1-7, the PLAYER car only gets
  engineForce *= 1 + 0.15*(7-round)/7 (+15% at L1, tapering to 0 by L8) so the first rounds feel
  forgiving. AI cars and the class physics baselines are untouched.

AI
- 8-10 AI drivers per field following the racing line with per-track difficulty + variance,
  drafting, slide jobs, inside defense, and pace bobbles. Sensible, varied, shuffling pack racing.

CARS — must look PICTURE-PERFECT (hard visual bar)
Every car (player and AI, both classes) must read clean at all times: four tires at the corners,
wings/spoiler on, body/livery intact, driver helmet, roll cage, no missing/floating/clipping parts.
Build procedurally. Winged sprint (Car.ts): body tub/tail/nose; a big raked TOP WING (cambered
leading lip, tall lettered side boards with number + sponsor text, wickerbill); smaller front wing;
the exposed sprint front end (round front NERF-BAR nose hoop + tubular straight FRONT AXLE on radius
rods); roll cage, nerf bars, engine block, chrome swept-up "zoomie" exhaust HEADERS; staggered
wheels with a big right-rear. Late model (LateModel.ts): the full-fendered wedge described under CAR
CLASSES. BOTH: build each tire by REVOLVING a cross-section profile (lathe) so it has rounded
shoulders and a slightly square tread — no sidewall or shoulder mesh may exceed the tread radius (a
too-big torus bulge reads as Mickey-Mouse ears). Lettered HOOSIER sidewalls on a chrome DISHED
beadlock wheel with a center nut. Set the tire material backFaceCulling = false. PBR paint with
clearcoat; per-side liveries drawn on canvas DynamicTextures (mirror text for the left side).
Distinct color + number per grid slot, with the AI cars carrying WING-TOP NUMBERS and an
ALWAYS-PRESENT red/white #42 in the field. Hero car = Super Jay's orange #32. AFTER ANY change to car
building / wheel placement / spawning, SCREENSHOT THE FULL GRID (and ?class=latemodel) and verify.

WORLD SCALE (the rule that prevents the recurring sizing bug)
Only the CARS and the TRACK are 1:10 scale. EVERYTHING ELSE is FULL REAL-WORLD size — marshals,
flag girl, drivers'-stand spectators, the lawnmower rider, the drivers' stand, the timing booth,
and any future person/prop/building. Working metric: 1 game unit ≈ 1 foot. Targets: standing adult
≈ 5.7u; drivers'-stand deck ≈ 5u; shack/booth ≈ 9u; a 1:10 car ≈ 2.5u long. People read ~2x a
car's height and clearly TOWER over the toy cars. Build procedural figures feet-anchored (root at
ground, y=0) and scale the root (scale = 5.7 / nativeHeight).

INPUT (gamepad primary, keyboard fallback, + sim rig + touch)
- Keyboard: arrows/WASD drive (steer only with auto-throttle on), R reset, V cycle camera,
  C quick-aerial, P pause, mouse-wheel / +/- zoom, G garage, M mute, K/J rig calibrate.
- Standard gamepad: axis0 steer, RT/LT (buttons 7/6) throttle/brake.
- SIM RIG (Logitech Flight Yoke + CH Pro Pedals show up as gamepads): SELF-CALIBRATING — learn each
  device's resting axis values, classify a centered-rest axis as steering and extreme-rest axes as
  pedals (polarity auto-detected). Only switch away from keyboard once the rig is actually moved.
- MOBILE: on-screen touch controls (steer pad + GAS/BRAKE/RESET + zoom +/-); GAS/BRAKE are HIDDEN
  when AUTO-THROTTLE is on (steering only).

RENDERING / REALISM
- Environment.ts: prefiltered IBL (.env in public/env), ACES tonemapping, bloom, SSAO2, SkyMaterial
  dome, light dust fog, FXAA + sharpen. DAY: bright blue sky, full IBL + sun. NIGHT (rounds 8/12/15):
  dark sky/fog, lower IBL, a brighter cool MOONLIGHT sun + ambient, a crescent MOON (billboarded
  emissive disc with an offset punch-out) + a STARFIELD dome (emissive random-dot texture,
  applyFog=false, emissive > 1 so bloom catches it) + a BIG DIPPER asterism (billboarded bright dots,
  north-up, pointer stars toward Polaris), and 6 lit LIGHT TOWERS (four corners + two mid-straight;
  PointLights). Select the career round BEFORE setting up the environment so lighting matches.
- ADAPTIVE QUALITY (QualityManager.ts): auto-scale render detail to hold ~60 fps via a 5-tier ladder
  (hardware scaling / MSAA / SSAO / bloom / sharpen) with hysteresis + a cooldown, sampling
  engine.getFps(). setupEnvironment returns EnvHandles { pipeline, ssao }; main.ts drives quality.update()
  each frame (window.__quality test hook). Tiers MULTIPLY onto the base hardware-scaling level so the
  lighter mobile base is preserved; shadow map size stays fixed.
- Cap shadow-map cost (refreshRate = 2, every other frame) — most casters are static.
- Dust: a ParticleSystem per car. Babylon defaults to ADDITIVE blend (glowing embers) — set
  BLENDMODE_STANDARD so it reads as opaque kicked-up dirt; tint from the track's dirtColor.
- Avoid ground MOIRÉ at grazing angles: anisotropicFilteringLevel = 16 on dirt textures, keep
  tiling modest, keep normal-map level low (~0.85). Camera maxZ must exceed skybox + star-dome size.

VERIFICATION (the user judges on look and feel)
- Run the dev server and use a headless browser to screenshot and read the console (zero errors).
  NOTE headless renders at ~1-5 fps, so the dt clamp slows the sim — to test physics/laps, step the
  sim SYNCHRONOUSLY at fixed dt in page script and read internal state (vehicle.heading,
  vehicle.position), not mesh.getDirection() (stale until a render frame). Expose window.__field,
  __track, __race, __cockpit, __audio.
- Per milestone: screenshot the scene (+ the FULL GRID for both classes), confirm the behavior,
  check FPS budget (target ~60, acceptable 40+ on a laptop dGPU), 0 console errors.

DISTRIBUTION (cross-platform)
- Live: https://aaronjblair.github.io/RC-Dirt-Oval/ (GitHub Pages, auto-deploys on push to main).

  game including the Havok .wasm, so it installs as a real app and runs offline on iOS/Android/
  Windows/Mac (iOS: Share -> Add to Home Screen; Android/desktop: the browser Install button). Icons
  generated by scripts/gen-icons.mjs -> public/pwa-192.png / pwa-512.png / pwa-maskable-512.png /

  build:win` runs scripts/build-win.mjs (builds dist/ then packages from a TEMP dir to dodge the

  Release asset). The `rebuild` skill orchestrates docs -> build -> executables -> publish -> links.
- Native iOS/Mac/Android store builds are intentionally NOT produced — install the PWA from the URL.
