# CLAUDE.md

## What this is
RCSprint вАФ a browser 3D 1/10-scale dirt-oval RC sprint car game modeled on the **Team Losi 22S Sprint**. Stack: **Babylon.js 7 + Havok (WASM) + Vite + TypeScript**. Driver-stand camera, sim-leaning physics, 15-track career. **Car classes:** the player picks a class on the start screen вАФ **Winged Sprint Car** or **Dirt Late Model** вАФ each with its own body, physics baseline, and **independent career** (`src/car/CarClass.ts`). **Audio:** subtle procedural **electric-motor whine** вАФ a detailed voice for the *player* car plus a lighter, **stereo-panned, distance-faded** whine for **every AI car** (Web Audio, brushless-RC style вАФ pitch tracks throttle/speed), all under one master gain; mute with **M** or the HUD рЯФК button (persisted).

**Two game modes** (`src/game/Mode.ts`, picked at start after class select): **Career/Sim** (the championship described above вАФ always-advance, points-keyed) and **Arcade** (RC Pro-Am style вАФ on-track pickups/letters/boost-strips/slicks, a score, and a top-3-or-burn-a-continue gate). **Both modes** start each race with a drag-strip **light tree** (staging dots вЖТ three ambers вЖТ GREEN) in place of the old 3-2-1 text countdown, with a **perfect-launch** boost for hitting the gas right on green.

## Hard rules
- **It ships.** `npm run build` вЖТ `dist/` must build clean and run from any static host. Use procedural assets or files under `public/`; no server-side dependencies.
- **The game is NIGHT.** RCSprint ships at **night, game-wide** вАФ `def.night = true` is forced in `main.ts` (dark sky, crescent moon + starfield, lit lamp towers). The night look is the game's identity. **Do NOT ship daytime racing.** (This happened once вАФ a "restore the day/night calendar" change made most rounds bright daylight, and it was reverted.) `?day` is a **dev-only** preview override; it must never be the shipped default.
- **All cars must look picture-perfect** вАФ every car (player and AI) reads as a clean winged sprint car: four corner tires, wing on, body/livery intact, nothing missing/floating/clipping.
- **World scale: ONLY the cars and the track are 1:10.** Every person/prop/building (track marshals, flag girl, drivers'-stand spectators, lawnmower rider, the drivers' stand, the timing booth) is **full real-world size**: 1 unit вЙИ 1 ft, a standing adult вЙИ **5.7u**, the stand deck вЙИ **5u**, a shack/booth вЙИ **9u**, a car вЙИ **2.5u**. People tower ~2√Ч a car's height. Scale procedural figures at the **feet** (root origin y=0). See the **world-scale** skill вАФ this rule exists because sizing kept getting redone.
- **Verify on screen, don't just describe.** Screenshot (full grid when touching `Car.ts`, `RaycastVehicle.placeWheels`, or spawning) or read sim state before calling a visual/physics change done вАФ see the **screenshot-game** skill.

## Commands
```
npm install
npm run dev      # Vite dev server at http://127.0.0.1:5173
npm run build    # tsc --noEmit (strict) then vite build -> dist/
npm run preview  # serve the production build
```
`npm run build` is the only gate (no test runner/linter). `tsconfig` is strict вАФ `noUnusedLocals/Parameters/noImplicitReturns`, so an unused symbol fails the build. Use `npx tsc --noEmit` for a fast typecheck.

In-game: arrows/WASD drive, R reset, **V (or the upper-left button) cycle camera view (In-Car/Track/Aerial)**, C quick-toggle aerial, G garage/setup, M mute engine sound. Gamepad/yoke+pedals take over on actual input.

## Architecture
- `src/main.ts` вАФ entry point + game-flow state machine (`attract вЖТ prerace вЖТ racing вЖТ finished`); boots engine, loads the career round, builds track/field, runs the fixed-timestep loop (`FIXED = 1/60`, вЙ§6 catch-up steps). Exposes `window.__field`, `__track`, `__race`, `__marshals` for tests. **Boot/loading progress bar:** the `#loading` splash is a real **app-build progress bar** (`#loadFill`/`#loadLabel`) driven by `setBootProgress(pct, label)` at staged points (engine 10% вЖТ physics 20/50% вЖТ track/field 85% вЖТ ready 100%), then fades out. **Start prompt:** clicking START opens `Screens.namePrompt` (pre-filled with the saved player name, default **"Super Jay"**); the typed name is title-cased (`titleCaseName`), persisted (`savePlayerName` вЖТ `localStorage["rcsprint.playername"]`), and set on the player racer (`player.name`) before the countdown. **Attract reel:** on first open per tab session (`sessionStorage rcsprint.seen`) all cars are AI-driven (`Field.attractUpdate`) under a cutting `CinematicCamera`; any click/key sets the flag and reloads into the menu with a fresh grid. Career **always advances** to the next (harder) round вАФ no podium gate.
- **Game modes** (`src/game/Mode.ts`): the player picks **Career/Sim** or **Arcade** at start (after class select). `loadMode`/`saveMode` persist the pick (`localStorage["rcsprint.mode"]`, `?mode=career|arcade` override); arcade run-state (`{round, continues, score}`) lives in `localStorage["rcsprint.arcade"]`. Start flow: class select вЖТ **mode select** (`Screens.modeSelect`) вЖТ pre-race menu; switching either persists + reloads. **Both modes** replace the 3-2-1 text countdown with a drag-strip **light tree** (`Screens.arcadeLightTree`: staging dots вЖТ three ambers вЖТ GREEN, "GET READY"/"GO!"), firing the green flag at the same ~2.4s moment; a **perfect-launch boost** (gas within ~350ms of green вЖТ a brief accel buff, `main.ts`). **Career/Sim is unchanged** вАФ always-advance, championship points. **Arcade** (`src/game/Arcade.ts` = `ArcadeManager`) adds on-track items on the oval: **7 PICKUPS** (player-only temporary buffs вАФ grip / accel / top-speed, plus a roll-cage immunity), **3 BOOST STRIPS** (any car вАФ brief speed/accel kick), **8 collectible LETTERS spelling "RCSPRINT"** (player-only; all 8 вЖТ big combined buff + 1000-pt bonus + upgraded flag), and **oil/wet SLICK patches** (any car вАФ transient grip loss вЖТ slides). Score accrues (pickups +100, letters +250). Arcade advancement requires a **top-3** finish to move on; otherwise burn one of ~3 **continues** вАФ out of continues resets the run. `finalize()` in `main.ts` branches by mode. Arcade HUD (`#arcadeHud` in `index.html`, Score / Continues / Letters) shows only in arcade mode; `window.__arcade` exposed for tests.
- **Temporary buff layer on the vehicle** (`src/physics/RaycastVehicle.ts`): `applyBuff(kind:"grip"|"accel"|"top", mult, seconds)`, `grantImmunity(seconds)`, and `buffState()` вАФ ticked/decayed in `update()` and woven into the grip/accel/top-speed math; immunity short-circuits `triggerRollover`. Behavior is identical when no buff is active (so Career/Sim is untouched).
- **The vehicle is custom and kinematic вАФ NOT a Havok body.** `src/physics/RaycastVehicle.ts` integrates its own velocity (slip/friction-circle tire model) + yaw and raycasts the ground for ride height + banking. Havok (`src/physics/PhysicsWorld.ts`) is used **only** for static track collision and wheel rays. Don't make the car a Havok rigid body вАФ that path was abandoned (applyForce desynced velocity from the mesh). Tune via `VehicleConfig`/`DEFAULT_CONFIG` and the slip/grip math.
- Car-to-car contact and wall limits are **positional**, not physics вАФ `src/race/Field.ts` (`resolveContacts`, `wallLimit`), which also owns surface grip, tire wear, dust, and rollovers for the whole field.
- **Rollovers are marshal-gated.** A hard hit (`RaycastVehicle.triggerRollover`) tumbles the car then leaves it **stuck upside down** (`isStuck`, no control/velocity) until a marshal reaches it; `R` (`resetTo`) clears it for the player. `src/race/Marshals.ts` places **6 full-size track marshals**: **2 SEATED in camp chairs at the two infield ENDS** (one chair per end, at x=0; `SIT_DROP` lowers them to read as seated) and **4 STANDING OUTSIDE the track at the four corners** (quarter-turn `cornerS` points offset `W/2 + 4` outward, facing in). Each marshal carries a `seated` flag so it returns to the right posture. Marshals have **varied looks** (`marshalLooks()`/`Look` вАФ different shirt/pants/skin/hair/headwear) so no two read identically (`buildPerson`). Figures are **rigged** вАФ `buildPerson` builds legs on **hip+knee pivots** and arms on **shoulder pivots** (rest pose reads identical to a plain standing figure), exposed via the `personRigs` WeakMap; a marshal **animates a full jog cycle** (legs swing from the hips, knees bend, arms counter-swing) while moving to/from an incident, and returns to rest when standing/seated/working. Every figure has **visible knee joints, shoes (soles at ground), and hands**, parented to the hip/knee/shoulder rig pivots so they swing with the gait вАФ this propagates to all marshals and drivers'-stand spectators; the bespoke figures (flag girl, lawnmower rider) got matching additions (the flag girl's wave-hand rides her flag pivot). World scale is unchanged (~5.7u tall, feet at yвЙИ0). See the **people-anatomy** skill. Static figures (spectators, the streaker's chaser) just leave the rig at rest. Updated each frame from `main.ts` with `field.cars`. They wait until trouble: the **nearest available** marshal (seated or standing) claims a car that is **wrecked (`isStuck`) OR stalled** (speed `< STALL_SPEED` for `> STALL_TIME` while green вАФ tracked in a per-car map), gets up/jogs to it, and **places it back on the racing line** вАФ upright, facing race direction вАФ via `vehicle.resetTo(pos, yaw)` from `track.project()` (the bottom groove), then returns to its post and re-seats/stands. (So marshals reposition onto the track, not just right in place.)
- **A flag girl starts each race.** `src/race/FlagGirl.ts` builds a procedural starter at the start/finish line (`track.sampleAt(0)`); `flagGirl.greenFlag()` (fired from the countdown GO and the `?demo` start) triggers a big green-flag wave that decays back to idle. Built/updated from `main.ts` like the marshals.
- **Audio is procedural Web Audio.** `src/audio/MotorSound.ts` synthesizes a subtle electric brushless-RC whine вАФ a sawtooth motor fundamental whose pitch tracks `speed`+throttle, a detuned higher **ESC/PWM "scream"**, a sub for body, and a faint speed-scaled tire/dirt noise вАФ through a low-pass that **opens with throttle** (keep the cutoff high enough to stay audible вАФ a too-low cutoff once silenced it: *re-open the filter*). All under a low master-gain cap so it stays subtle. The `AudioContext` starts **suspended**; `main.ts` calls `motor.resume()` on the first `pointerdown`/`keydown` (browser autoplay), then `motor.update(throttle, speed)` each physics step **only in the racing state** (the detailed player voice). The **AI field** gets a cheaper tier: `motor.setVoiceCount(n)` makes one lightweight saw-oscвЖТgainвЖТ**stereo panner** voice per other car (no filter/sub/noise), and `motor.updateVoices(...)` each frame pitches/pans/distance-fades them to the active camera вАФ a subtle pack, not a swarm. Mute via **M** / the `#mute` HUD button вЖТ `setMuted` (master gain 0 mutes everything, persisted to `localStorage["rcsprint.muted"]`, re-read on load). Exposed as `window.__audio`. No audio file ships. Degrades to a silent no-op if `AudioContext` is unavailable (headless).
- **Car classes** (`src/car/CarClass.ts`): the player races a **Winged Sprint Car** (`createCar`/`DEFAULT_CONFIG`) or a **Dirt Late Model** (`src/car/LateModel.ts` `createLateModel`/`LATE_MODEL_CONFIG` вАФ heavier, planted, full-fendered). `CAR_CLASSES`/`CAR_CLASS_LIST` map each id to its body builder + pristine physics baseline; `loadCarClass`/`saveCarClass` persist the pick (`localStorage["rcsprint.class"]`, `?class=sprint|latemodel` override). `Field` takes a `CarClassDef` and builds every car via `classDef.build`. Careers are **class-keyed** (`Career.ts` `loadCareer(cls)`/`saveCareer(c, cls)` under `rcsprint.career.<cls>`, with a one-time migration of the old single-class save into `sprint`). The start screen shows `Screens.classSelect` first; switching class persists + reloads so the field/career rebuild. `SetupPanel` shows the class label. A hidden **`?photo`** mode locks a close rear-3/4 camera onto the player car (shares the post-FX) to actually SEE the bodywork. **Opening splash:** the `#loading` screen shows the real **SUPER JAY #32** racing badge (`public/superjay-32.png`) above the build progress bar; the original shop photo lives in `src/assets/superjay-photo.jpg`.
- **Easter egg:** `src/race/LawnMower.ts` builds a guy on a red riding mower parked on the infield grass by the logo (static, built from `main.ts`).
- **The car is modeled on a real winged sprinter** (`src/car/Car.ts`): a huge top wing with a **down-swept front scoop** + tall number side boards, **staggered tires** (biggest on the right-rear) on **orange beadlock** wheels, and a detailed tube front end. Tire stagger needs **per-wheel radius** вАФ `WheelDef.radius` feeds `RaycastVehicle.placeWheels` (each tire sits on its own radius); without it all wheels use `cfg.wheelRadius` and big tires sink/float.
- Tracks are data (`src/track/TrackDef.ts`); `OvalTrack` builds the banked oval (plus a **grassed infield** filling the oval with the **Flora Vista speedway logo sprayed flat onto it** вАФ `src/assets/logo.png`, imported as a bundled Vite asset, alpha-blended with a faint emissive so it reads as bold paint; sized large to most of the infield, its long axis running along the straights/z). `src/track/tracks.ts` `generateCareer()` produces the 15 rounds (night rounds 8/12/15, but `main.ts` currently **forces night game-wide** via `def.night = true`, and **randomizes the field to 8вАУ12 cars per race** via `def.fieldSize = 8 + rand(0..4)` вАФ so `PALETTE` (Field.ts) and `DRIVER_NUMBERS` (Career.ts) carry 12 entries). **Driver names:** slot 0 is the player (their saved name, default **"Super Jay"**); AI slots draw a **stable full name per grid slot** from `AI_NAMES` (Career.ts) so the championship (points keyed by name in `awardPoints`) stays coherent round-to-round вАФ `driverName(i)` (used by `Field.add`) is deterministic, no `Math.random`. `Scenery.ts` builds trackside (real-size **drivers' stand** with **varied full-size spectators** вАФ they reuse the marshals' `buildPerson`/`spectatorLooks` so they have arms, varied shirts, some caps, some long hair; the spectators are **yawed to face the track** (вИТx, toward the oval center) with arms posed **forward** via the exported `personRigs` shoulder pivots, **each holding a small procedural RC transmitter** prop (box body + thumbstick nubs + antenna) built between the hands, a **timing booth/shack** with a dark-gray roof on the +z end of the stand, **6 light towers** = 4 corners + 2 mid-straight, start/finish) plus a **per-round themed backdrop** (`TrackDef.backdrop`: mesas/forest/plains/city/dunes/badlands) and a large **world floor** under it (the 400m infield only covers ¬±200m, so distant scenery needs ground out to the horizon). Career save in `src/career/Career.ts` (localStorage); car setup in `src/car/CarSetup.ts`.
- **Three camera views** (selected each frame in `main.ts` from a `view` enum `"normal"|"incar"|"aerial"` that **always starts at `"normal"`/Track each race**, with a `?view=incar|aerial` override): the `DriverStandCamera` follow cam ("normal"/Track), the `aerial` `UniversalCamera`, and **`src/core/CockpitCamera.ts`** вАФ a first-person in-car camera **parented to the player car root** (`field.cars[0].root`) via a `cockpitEye` node, so it inherits heading/pitch/roll/bank; `update()` adds subtle lean-into-corner (from yaw-rate √Ч speed), a faint speed shake, and a speed-FOV. New cameras must be `env.pipeline.addCamera(...)`'d (ACES/bloom parity) **and** attached to the `"ssao"` pipeline (`scene.postProcessRenderPipelineManager.attachCamerasToRenderPipeline`). Upper-left `#view` button + `V` cycle the views (`C` still quick-toggles aerial); a flip (`isStuck`/`isRolling`) falls the in-car view back to the external cam. Exposed as `window.__cockpit`.
- Rendering: Babylon imported **√† la carte** with required **side-effect imports** (materials, shadow/physics components, prepass+geometryBuffer for SSAO2). `src/core/Environment.ts` `setupEnvironment` sets IBL/ACES/bloom/SSAO/SkyMaterial (day + night) and **returns an `EnvHandles { pipeline, ssao }`** (the SSAO2 pipeline may be `null` if its try/catch failed on a weak GPU); its `addNightSky()` adds a **crescent moon** (billboarded carved disc) + a **starfield dome** + a **Big Dipper** asterism (billboarded bright dots, north-up, all `applyFog=false`) at night. Dust/dirt are procedural canvas textures (`src/core/Textures.ts`). The racing surface is **one uniform packed-dirt brown** (no painted groove bands; grip still evolves invisibly in `SurfaceModel`).
- **Adaptive graphics quality** (`src/core/QualityManager.ts`): auto-scales render detail to hold ~60 FPS, climbing to max when the GPU has headroom. A **5-tier ladder** (0 Min вА¶ 4 Ultra) controls hardware scaling (render resolution via `engine.setHardwareScalingLevel`), MSAA samples, SSAO samples, bloom, and sharpen вАФ each tier's `scaleMul` **multiplies the base** hardware-scaling level so the existing mobile lighter-render path is preserved (mobile just starts/sits lower). `main.ts` constructs `new QualityManager(engine, env.pipeline, env.ssao, coarsePointer ? 1 : 3)` (start tier **High 3** desktop / **Low 1** mobile) and calls `quality.update(engine.getDeltaTime())` **every frame** (not gated by race state). The controller samples `engine.getFps()` every 0.5s with hysteresis (step DOWN if <50 fps sustained ~1s, UP if >58 fps sustained ~3s) and a 2s cooldown. Exposed as `window.__quality` (`{ tier, max, setTier, update }`) for tests. **Shadow-map size stays fixed** (not runtime-mutable вАФ never touch it).
- **Early-career player speed boost** (`main.ts`, right after the field is built): for career levels 1вАУ7 the **player car only** gets `engineForce *= 1 + 0.15*(7-round)/7` (round 0-based) вАФ +15% on level 1 tapering to 0 by level 8. AI cars and the per-class baselines are untouched. Class-agnostic (top speed вЙИ engineForce/rollResist). See the **round-speed-boost** skill.

## Gotchas (cost real time)
- **Headless sim is slow (~1вАУ5 fps), so render-loop timing misleads.** For deterministic tests, step the sim **synchronously at fixed dt** in `browser_evaluate` and read internal state (`vehicle.heading`, `vehicle.position`) вАФ not `mesh.getDirection()`, which is stale until a render frame. (Adding assets/actors? see the **import-asset** / **add-trackside-actor** skills.)
- **Backdrops barely show in the racing/driver-stand POV** вАФ that elevated camera follows the car and pitches down into the bowl, so distant horizon silhouettes sit above the frame. They read in the **aerial (C) / attract wide** views; verify backdrop changes there (attract reel via `?round=N`), and keep silhouettes tall/close enough to clear the horizon haze.
- **Backdrop instances must never reach inboard of the outfield** (a wide butte/mesa base sitting at the ring radius will overlap the racing surface and "cut off" the track). `buildBackdrop` clamps each instance's radius with `safeR(r, footprint)` to push it out by half its own width; a footprint is closest to the track along **x** (the z ring is stretched √Ч`zS`). Trim base/height multipliers rather than relying only on distance.
- **Night moon/stars sit ABOVE the bowl-camera frame** (same reason backdrops barely show). Don't conclude "not rendering" from a `?demo`/attract shot вАФ verify by freezing the scene (`scene.onBeforeRenderObservable.clear()`), aiming a camera at `moon.position`, `scene.render()`, then a Playwright screenshot (`maxZ` must exceed the star-dome radius). See the **night-sky** skill.
- **Don't eyeball figure scale from the racing cameras** вАФ they distort size. Probe a figure's world bounding-box height via `browser_evaluate` (expect a person вЙИ5.7u, feet yвЙИ0). See the **world-scale** / **verify-goals** skills.
- A **Logitech Flight Yoke + CH Pro Pedals** show up as gamepads; a held button can hijack the keyboard. For deterministic tests, override `navigator.getGamepads = () => []`.
- Babylon `ParticleSystem` defaults to **additive blend** вАФ dust looked like glowing embers until set to `BLENDMODE_STANDARD`.
- **Ground moir√©** at grazing angles = over-tiled dirt; use `anisotropicFilteringLevel = 16`, modest tiling, lower `bumpTexture.level`.
- `Matrix.InvertToRef` is not static вАФ use `matrix.invertToRef(out)`.
- Camera `maxZ` must stay above the skybox size or the sky clips to black.
- **Finishing order must rank finished cars by FINISH TIME, not by `progress`.** A finished car's `progress` (`lap*len + s`, `RaceManager`) wraps DOWN at the line (sвЖТ0, lap clamped to `totalLaps`), so a progress-only sort ranks the actual winner *behind* cars still on the last lap вАФ the P1 finisher showed as P2+. Sort finished cars first by `finishedAt`, then unfinished by `progress`.
- **Tires/wheels: no sidewall or shoulder mesh may exceed the tread radius.** A torus "bulge" sized past the tread reads as Mickey-Mouse ears. Revolve the tire carcass from a cross-section with `MeshBuilder.CreateLathe` (rounded shoulders, square-ish tread) and set the tire material `backFaceCulling = false` (a revolved surface is single-sided). Hoosier lettering goes on a thin sidewall disc with a punched-out center; the chrome **dished** wheel + center nut shows through.
- **Mobile touch controls must set `touch-action:none` on EACH interactive element** (`Input.ts` `setupTouch` вАФ pad/knob/GAS/BRAKE/RESET), not just the `#touchControls` root: `touch-action` is **not inherited**, and the root is `pointer-events:none` so the touches land on the children. Miss it and a **second finger** (steering while on the gas вАФ happens mid-corner) lets the browser pinch/pan the visual viewport (iOS ignores `user-scalable=no` for zoom), scrolling the `position:fixed` buttons out of view вАФ they "disappear" вАФ and firing `pointercancel` that drops throttle. Also bind `lostpointercapture` so a revoked capture always releases (no stuck/unresponsive button).
- **electron-builder EPERM on the project tree** вАФ building the Windows installer straight into `./release/` under the project dir trips an **EPERM from a filesystem filter** on the tree. The workaround (`npm run build:win` вЖТ `scripts/build-win.mjs`): redirect electron-builder's output to a **temp dir**, then copy the finished installer into `./release/`. The `.exe` ships as a **GitHub Release asset**, never committed (`release/` is gitignored).
- **`touch-action:none` + `gesturestart` preventDefault are NOT enough to stop iOS Safari pinch-zoom** when each finger is captured by a control вАФ it still zoomed in real use. The **reliable** block is canceling any **multi-touch** `touchmove`: `document.addEventListener("touchmove", e => { if (e.touches.length > 1) e.preventDefault(); }, {passive:false})` (single-finger scroll stays intact, so the Guide overlay/name input still work). Belt-and-suspenders: **pin `#touchControls` to `window.visualViewport`** (counter-`translate`+`scale(1/scale)` + size to `vv.width/height` on its `resize`/`scroll`) so the controls track the visible area and can never scroll off-screen even if a zoom slips through or the Safari toolbar reflows.

## Distribution
The game ships **two ways** (see `DISTRIBUTION.md` / `README.md`'s Install / Download section):
- **Installable PWA** via `vite-plugin-pwa` вАФ the build emits a `manifest.webmanifest` + service worker that **precaches the bundle incl. the Havok `.wasm`**; icons are generated by `scripts/gen-icons.mjs` from `public/superjay-32.png` (`pwa-192.png`/`pwa-512.png`/`pwa-maskable-512.png`/`apple-touch-icon-180.png`). Installs as an app on **iOS, Android, Windows, and Mac** from the browser. Live: `https://aaronjblair.github.io/RCSprint/`.
- **Windows installer (.exe)** via Electron + electron-builder вАФ `electron/main.cjs` loads the built `dist/index.html`; config in `electron-builder.yml`; built with `npm run build:win` (вЖТ `scripts/build-win.mjs`). Ships as a **GitHub Release asset**, never committed.
- Native iOS/Mac/Android **store** builds are **intentionally not produced** (no Apple hardware/account; iOS can't sideload) вАФ install the PWA instead.
- The **rebuild** skill orchestrates the whole release (docs вЖТ build вЖТ executables вЖТ publish вЖТ links).

## Windows / PowerShell
- `npm` is `npm.cmd` вАФ `Start-Process "npm"` fails. Run `npm run dev` as a background process instead.
- PowerShell 5.1 has no `&&`. Chain with `;` or `if ($?) { ... }`.
- **Git is installed but NOT on PATH.** Prepend it before any git command: `$env:Path = "C:\Program Files\Git\cmd;" + $env:Path`. Use `git --no-pager вА¶` for log/diff. The repo's `origin` is the **public** GitHub repo `aaronjblair/RCSprint` (auto-deploys to GitHub Pages on push); commit + push to `main` via the **commit-it** skill.
